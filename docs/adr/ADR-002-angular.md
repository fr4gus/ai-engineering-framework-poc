# ADR-002 — Angular

## Decision

Use Angular as the frontend framework.

## Why I recommend Angular

Based on what I know about your background:

You have extensive software architecture experience.
You have significant Android development experience.
You value maintainability and strong project structure.
This project emphasizes documentation and long-term organization.

Angular aligns well with those goals because it provides:

Opinionated architecture.
Dependency Injection.
Excellent TypeScript support.
Clear module boundaries.
Strong tooling.
Built-in routing.
Signals (modern state management).
Standalone components.

For an AI-driven workflow, Angular is also advantageous because the project structure is highly predictable, making it easier for planning and implementation agents to navigate.
